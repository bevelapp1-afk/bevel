# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

ClearView ERP is a single-file, zero-dependency HTML application for glass shop business management. The entire app lives in `glass-erp.html` — CSS in `<style>`, markup in `<body>`, and JavaScript in a single `<script>` block at the bottom. There is no build step, no package manager, and no backend.

To run: open `glass-erp.html` directly in a browser.

## Architecture

### Data Layer (`DB` object, line ~745)

All persistence is `localStorage` with a `cv3_` key prefix. The `DB` object is the sole data access layer:

- `DB.clients()`, `DB.projects()`, `DB.estimates()`, `DB.invoices()`, `DB.attachments()`, `DB.audit()` — each returns a parsed array from `localStorage`
- `DB.settings()` / `DB.setSetting(obj)` — settings stored separately as `cv3_settings` (object, not array)
- `DB.get(k)` / `DB.set(k, v)` — raw read/write with prefix

IDs are generated by `uid(prefix)` — `prefix + '_' + timestamp + '_' + random`.

### Navigation

`showPage(name, opts)` is the router. It toggles `.active` on `.page` divs (matched by `id="page-{name}"`) and `.nav-item` buttons (matched by `id="nav-{name}"`). Two globals track drill-down state: `currentClientId` and `currentProjectId`.

Detail pages (`client-detail`, `project-detail`) are separate `.page` divs, not modals — they're shown via `showPage('client-detail', {clientId: id})`.

### Pipeline & Statuses

`PIPELINE_STAGES` (line ~1677) is the source of truth for project statuses. It drives:
- The kanban board columns and list groupings in the Pipeline view
- Status badge colors everywhere (via `getStatuses()` → `statusBadge(label)`)
- The status editor in Settings (customizable, falls back to `PIPELINE_STAGES` defaults)

### Modals

All modals use the same pattern: `openModal('modal-id')` / `closeModal('modal-id')` toggle a CSS `.open` class. Each modal has its own `open*Modal(id?)` function that pre-populates fields for edit vs. create, and a corresponding `save*()` function that writes to `DB`.

### PDF / Print Export

`printPDF()` calls `window.print()`. The `@media print` CSS section (~line 190) hides the sidebar/topbar and exposes the `#pdf-content` div, which is populated by `buildPDFContent()` before printing.

### Email

Emails are sent via `window.location.href = 'mailto:...'` — no mail API. The send-document modal composes a `mailto:` URL with subject and pre-filled body.

## Key Patterns

- **Render functions** (`renderClients()`, `renderProjects()`, etc.) do full re-renders of their section by setting `.innerHTML`. Call these after any mutation to that collection.
- **Currency formatting**: always use `$$(amount)` — never format manually.
- **Sequential numbering**: `seqNum(prefix, array, startKey)` generates `EST-001`-style numbers, respecting the starting number from Settings.
- **Audit logging**: call `audit(action, detail)` after every data mutation; capped at 500 entries.
